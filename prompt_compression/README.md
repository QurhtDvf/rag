# Prompt Compression (MCP + FastAPI)

## Overview
本プロジェクトは、複数のプロンプト圧縮手法（compressor）を比較・評価し、最適な圧縮結果を用いてLLM推論を行うパイプラインを提供するものである。  
同一のコアロジックを、**MCPサーバー**および**FastAPI**の両インターフェースから利用可能とする構成になっている。


## Compression Methods

本システムでは、複数の異なるアプローチによる圧縮手法を併用し、評価によって最適なものを選択する。

### 1. TF-IDF Compression
文書を文単位に分割し、TF-IDFスコアに基づいて重要度の高い文を抽出する手法である。  
軽量かつ高速であり、ベースラインとして機能する。

---

### 2. Selective Context
言語モデルのトークン重要度に基づき、情報量の低い部分を削除する手法である。  
文脈を保ちながら冗長性を削減することに強みがある。

---

### 3. LLMLingua
トークン単位で重要度を推定し、意味を維持したまま大幅に圧縮する手法である。  
特にプロンプト圧縮に特化して設計されている。

---

### 4. LongLLMLingua
LLMLinguaを長文向けに拡張した手法であり、文書全体の構造を考慮して圧縮を行う。  
長いコンテキストでも情報の偏りを抑える。

---

### 5. Attention-based Pruning
Transformerのattention重みを利用し、重要度の低いトークンを削除する手法である。  
モデル内部の情報を直接活用する点が特徴である。

---

## Strategy

これらの手法を個別に適用した後、以下の指標で評価する：

- ROUGEスコア（内容保持）
- embedding類似度（意味保持）
最終的に最もスコアの高い圧縮結果が選択され、LLMに入力される。

## Features

- 複数compressorの並列実行
